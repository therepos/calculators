# Praxis

Standalone HTML calculators and modelers. No build step, no dependencies — drop an `.html` file into `apps/` and it appears on the landing page.

## How it works

1. Add an `.html` file to `apps/`
2. Push to `main`
3. A GitHub Action scans `apps/`, extracts `<title>`, `<meta name="description">`, and `<meta name="category">` from each file, and writes `apps/manifest.json`
4. The landing page (`index.html`) fetches that manifest and renders the list

The index page updates automatically on every push. Filter chips are built from whatever categories exist in the manifest — no home-page edits needed.

## Adding a new tool

Create an HTML file in `apps/` with these tags in `<head>`:

```html
<title>My Calculator</title>
<meta name="description" content="Short subtitle shown on the landing page.">
<meta name="category" content="Finance">
```

`category` is free-form. New values (e.g. `Research`, `Legal`) become new filter chips automatically. Defaults to `Other` if omitted.

If your calculator defines a `--accent` CSS variable, the manifest records it for future use.

## Hosting

Served via GitHub Pages from the `main` branch. No server-side code needed.

## Structure

```
index.html                          Landing page (reads manifest.json)
apps/                               Calculator HTML files
apps/manifest.json                  Auto-generated by GitHub Action
shared/praxis.css + praxis.js       Shared nav bar injected into every tool page
.github/workflows/manifest.yml      Manifest generator
```
